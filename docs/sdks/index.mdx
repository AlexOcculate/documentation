Pieces provides SDK libraries for major programming languages like Python, Dart, Kotlin and TypeScript so you don't have to write code for interacting with our API protocols from scratch.

We're always working on improving and extending the current stack of available SDKs and SDK documentation, listed below is a list of official SDKS the Pieces team is maintaining.

## SDKs List
| Language                                              | Github Repository                                                                                                                      |
|-------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| [TypeScript](/sdks/typescript/guides/getting-started) | [https://github.com/pieces-app/pieces-os-client-sdk-for-typescript](https://github.com/pieces-app/pieces-os-client-sdk-for-typescript) |
| [Python](/sdks/python/guides/getting-started)         | [https://github.com/pieces-app/pieces-os-client-sdk-for-python](https://github.com/pieces-app/pieces-os-client-sdk-for-python)         |
| [Dart](/sdks/dart/guides/getting-started)             | [https://github.com/pieces-app/pieces-os-client-sdk-for-dart](https://github.com/pieces-app/pieces-os-client-sdk-for-dart)             |
| [Kotlin](/sdks/kotlin/guides/getting-started)         | [https://github.com/pieces-app/pieces-os-client-sdk-for-kotlin](https://github.com/pieces-app/pieces-os-client-sdk-for-kotlin)         |

## Community
If you have created your own SDK or any other technology specific integration and would like us to list it here under community maintained SDKs/integrations please contact us.

If you would like to help us expand Pieces' list of SDKs, you can start a new discussion on our [Open Source Discussions](https://github.com/pieces-app/opensource/discussions) and you can also [join our Discord](https://discord.gg/getpieces).
